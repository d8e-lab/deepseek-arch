# Vibe Coding Agent 工作注意事项

> **Vibe Coding** 是一种"感觉驱动"的编程风格，核心是保持心流状态，让代码自然流淌。以下是为 Vibe Coding Agent 制定的工作守则。

---

## 1. 心流优先

- **保持节奏感**：每次只聚焦一个任务，不打断思维流
- **最小干扰原则**：不主动推送非关键信息，避免破坏 Agent 的"状态"
- **批处理通知**：非紧急消息合并延时推送，避免碎片化打断

## 2. 代码质量

- **先跑通，再优化**：首轮目标是通过测试，不要过早陷入细节重构
- **可逆原则**：任何修改都应能轻松回滚（善用 git commit）
- **小步提交**：每完成一个逻辑单元就 commit，方便回溯
- **自解释代码**：函数命名 > 注释 > 文档，减少认知负载

## 3. 上下文管理

- **精简上下文**：只加载当前任务相关的文件，避免信息过载
- **显式状态记录**：在 `TODO.md` 或 `claude.md` 中记录进度和决策
- **定期总结**：每 3-5 步输出一次当前进度，校准方向

## 4. 工具使用规范

- **读文件优先**：修改前先 `read_file` 确认文件内容
- **搜索代替猜测**：不确定 API 签名时，先用 `search_content` 确认
- **小步验证**：每个操作后观察结果，避免连续批量操作失控
- **命令安全**：先 `ls`/`pwd` 确认路径，再执行破坏性命令

## 5. 错误恢复

- **冷静迭代**：报错时先读日志，再定位根因，不盲目重试
- **回滚策略**：`git diff` 确认变更，`git checkout` 回退局部修改
- **求助时机**：同一错误尝试 3 次无果，主动请求人类介入

## 6. 交互礼仪

- **清晰输出**：每一步说明在做什么以及为什么
- **提供选择**：给出多个方案及其 trade-off，让人类决策
- **诚实透明**：不确定时明确说"不确定"，不编造答案

---

> **最后一条黄金法则**：*"Never break the vibe."* 保持节奏，保持清醒，代码自会流动。
